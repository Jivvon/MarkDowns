# chapter05,06 데이터 타입, 클래스

#### 5.1 데이터 타입 분류

- 기본 타입 : 정수, 실수, 논리 타입
- 참조 타입 : 배열, 열거, 클래스, 인터페이스 타입

#### 5.2 메모리 사용 영역

- 메소드 영역, 힙 영역, JVM 스택

#### 5.3 참조 변수의 ==, != 연산

- 기본 타입 : 변수의 값이 같은지 다른지 비교
- 참조 타입 : 동일한 객체를 참조하는지 다른 객체를 참조하는지 조사

#### 5.4 Null & NullPointerException

- null : 변수가 참조하는 객체가 없을 경우 초기값으로 사용 가능하며, 참조 타입의 변수에만 저장 가능하다.
- NullPointerException
  - 예외의 한 종류이며, 참조 변수가 null 값을 가지고 있을 때, 객체의 필드나 메소드를 사용하려고 했을 때 발생

#### 5.5 String

- 문자열을 저장하는 클래스 타입
- 문자열 리터럴이 동일하다면 String 객체를 공유

#### 5.6 배열 타입

- 배열 : 같은 타입의 데이터를 연속된 공간에 저장하는 자료구조. 특정 데이터 저장 위치는 인덱스를 부여하여 접근할 수 있다.

- 중복된 변수 선언을 줄이기 위해 사용되며, 반복문을 이용할 시 효율이 극대화된다.

  ```java
  타입[] 변수;
  int[] intArr;
  타입[] 변수 = null; // 배열 변수는 참조 변수이므로 배열 생성 전 null로 초기화 가능
  ```

- new 연산자로 배열을 생성할 수 있는데, 생성시 값 목록을 가지고 있지 않는다.
- **배열 복사**
  - 배열은 크기 변경 불가
  - 더 많은 저장 공간이 필요하다면 보다 큰 배열을 새로 만들고 이전 배열로부터 항목 값들을 복사

#### 5.7 열거 타입

- 한정된 값만을 갖는 타입
- 참조 타입이므로 Null 가능

```java
public enum 열거타입이름 {...} // 첫 글자 대문자
```

#### 6.1 객체 지향 프로그래밍

- OOP: Object Oriented Programming
- 부품 객체를 먼저 만들고 하나씩 조립해 완성된 프로그램을 만드는 기법
- 캡슐화, 상속, 다형성

#### 6.2 객체와 클래스

- 클래스에는 객체를 생성하기 위한 필드와 메소드가 정의
- 클래스로부터 만들어진 객체를 해당 클래스의 인스턴스
- 하나의 클래스로 여러 개의 인스턴스 생성 가능

#### 6.3 클래스 선언

- 클래스이름.java (첫 문자는 대문자로)
- 소스 파일당 하나의 클래스를 선언하는 것이 관례

#### 6.4 객체 생성과 클래스 변수

- new 연산자로 객체 생성.
- 클래스 변수 : new 연산자에 의해 리턴된 객체의 주소 저장 (참조 타입)

```java
클래스 변수;
변수 = new 클래스(); // 생성자를 호출하는 코드
```
