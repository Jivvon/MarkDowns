# chapter06 클래스

#### 6.5 클래스의 구성 멤버

- 필드
- 생성자
- 메소드

#### 6.6 필드

- 필드의 내용 : 객체의 고유 데이터, 상태 등
- 필드 선언 : 타입 필드 [= 초기값];
- 기본 초기값은 수의 경우 0이고, char는 '', boolean은 false, 참조 타입은 null이다.
- 필드 사용 : 객체 내부에서는 "필드이름"으로 접근하고, 객체 외부에서는 "변수.필드이름"으로 접근한다.

#### 6.7 생성자

- 생성자 : new 연산자에 의해 호출되며, 필드의 값을 설정하고 메소드를 호출해 객체를 사용할 수 있도록 준비한다.
- 기본 생성자
  - 모든 클래스는 생성자가 **반드시 존재**하며 **하나 이상 가질 수 있다.** 선언을 생략하면 컴파일러가 기본 생성자 추가.
- 생성자는 다양화 해야한다. 객체 생성할 때 초기화할 외부값이 어떤 타입으로 몇 개가 제공될지 모르기 때문이다.
- 생성자 오버로딩 : 매개변수의 타입, 개수, 순서가 다른 생성자 여러 개 선언
  - 생성자 오버로딩되면 생성자 간의 중복된 코드가 발생할 수 있는데, 초기화 내용을 한 생성자로 몰아서 작성한 후, 다른 생성자에서 this로 호출하는 식으로 작성한다.

#### 6.8 메소드

- 메소드란 객체의 동작(기능)이다.
- 리턴은 메소드가 작동 후 반환하는 것이며, 매개변수는 메소드를 실행할 때 필요한 데이터를 외부에서 받기 위해 사용한다.
- 메소드는 클래스 내/외부의 호출에 의해 실행되며 외부에서는 생성된 클래스 객체의 참조 변수를 이용하여 호출한다.
- 메소드 오버로딩 : 클래스 내의 같은 이름의 메소드를 여러 개 선언. 
  - 하나의 메소드 이름으로 다양한 매개값을 받기 위함이며, 매개변수의 타입, 개수, 순서가 달라야 오버로딩이 가능하다.
