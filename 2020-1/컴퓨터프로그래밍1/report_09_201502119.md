# chapter06 클래스

#### 6.7 생성자

- new 연산자에 의해 호출되어 객체의 초기화 담당

  - ```java
    new [ClassName]();
    ```

- 기본 생성자

  - 모든 클래스는 생성자가 반드시 존재하며 하나 이상 가질 수 있다
  - 생성자 선언을 생략하면 컴파일러는 기본 생성자를 추가한다. (개발자가 생성자를 선언했다면 추가하지 않음)

- 생성자를 다양화해야 하는 이유 ?

  - 외부 값이 어떤 타입으로 몇 개가 제공될 지 모른다 -> 생성자 오버로딩(Overloading)

#### 6.8 메소드

- **함수**. 객체의 기능. 호출해서 실행할 수 있는 블록.
- 반환값과 매개변수을 특정할 수도 있고 없을 수도 있다
- 메소드 오버로딩 (같은 이름의 메소드를 여러 개 선언하는 것 like 생성자 오버로딩)
  - 조건 : 매개변수의 타입, 개수, 순서가 달라야 한다.

#### 6.9 인스턴스 멤버와 this

- 인스턴스 멤버란?
  - 객체(인스턴스)마다 가지고 있는 필드와 메소드
- this
  - 객체(인스턴스) 자신의 참조를 가지고 있는 키워드
  - 객체 내부에서 인스턴스 멤버임을 명확히 하기 위해 사용한다.

#### 6.10 정적 멤버와 static

- 정적(static) 멤버란?

  - 클래스에 고정된 필드와 메소드 (정적 필드/메소드)

  - **정적 멤버는 클래스에 소속된 멤버이다.** (객체(인스턴스) 내부가 아닌 메소드 영역에 존재한다)

  - ```java
    public class Calculator{
      static double pi = 3.141592;
      ...
    }
    Calculator.pi // 클래스이름과 함께 도트 연산자로 접근
    ```

  - 공용 데이터는 정적 필드로 선언한다

- 싱글톤(Singleton)

  - 하나의 어플리케이션 내에서 단 하나만 생성되는 객체
  - 생성자에 private 접근 제한자를 붙인다.
  - 외부에서 호출할 수 있는 정적 메소드인 getInstance() 선언

#### 6.11 final 필드와 상수(static final)

- final 필드는 값을 변경할 수 없다

- 상수 = 정적 final 필드

  - ```java
    static final double PI = 3.141592;
    ```

#### 6.13 접근제한자

- 클래스    : public, default

- 생성자    : public, protected, default, private

- 멤버변수   : public, protected, default, private

- 멤버메소드 : public, protected, default, private

- 지역변수   : 접근제한자 사용 불허

#### 6.14 getter와 setter

- 클래스 선언할 때 필드는 일반적으로 private 접근 제한
  - 읽기 전용 필드에는 getter가 필요하다
  - 외부에서 마음대로 변경하지 못하도록 setter가 필요하다

Classroom과 Student 클래스를 만들었다.

Classroom은 학생들이 포함되며 학생들은 Student 클래스의 인스턴스이다.

Student 클래스에는 학생 이름과 학생의 연락처를 전용 필드로 정의되어 있다.

각 필드의 setter와 getter를 만들었는데, 기본값을 따로 설정하지 않은 값(변경 불가능)은 setter를 만들지 않았다.

Classroom의 students 필드의 setter는 addStudents()와 subStudents()로 설정하였다.

Student 클래스의 setPhone() 함수는 수업(같은 패키지 내)에서 학생의 연락처를 변경할 수 있도록 protected로 구성하였다.

Classroom의 생성자는 3개, Student의 생성자는 2개로 만들었다. Classroom의 세 번째 생성자는 Student를 매개변수로 받아 수업을 만들자마자 한 명의 학생을 참가할 수 있도록 하였다.
