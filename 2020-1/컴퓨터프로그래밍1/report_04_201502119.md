# chapter02 변수와 타입

### 2.1 변수

프로그램은 작업을 처리하는 과정에서 필요에 따라 데이터를 메모리에 저장한다.

변수 : 값을 저장할 수 있는 메모리의 공간

변수에는 **한 가지 타입**의 값만 지정할 수 있다.

변수를 사용하기 위해서는 **타입**과 **이름**을 결정하여 선언한다.

- 타입 : 변수에 저장되는 값의 종류와 범위를 결정 짓는 요소.
- 이름 : 메모리 주소에 붙여진 이름.

```java
int a = 5;
```
**자료형**은 int(정수), **변수 이름**이 a인 변수에 **대입**(=)을 5(**초기값**)으로 **초기화**함과 동시에 **선언**하였다.

- 변수의 초기값은 코드에서 직접 입력하는 경우가 많은데 소스 코드 내에서 직접 입력된 값을 **리터럴**(literal)이라고 부른다.
  - 정수/실수/문자/논리 리터럴로 구분.
  - 리터럴은 상수(constant)와 같은 의미지만, 프로그램에서 이미 다르게 정의하고 있으므로 리터럴이라는 용어를 사용하여 구분한다.

- 변수는 초기화가 되어야만 읽을 수 있다.

- 변수는 중괄호 {} 블록 내에서 선언되고 사용된다 : 클래스, 생성자, 메소드

- 메소드 내에 선언된 변수를 로컬(지역) 변수(local variable)라고 부른다.

  - 로컬 변수는 메소드 실행이 끝나면 메모리에서 자동으로 없어진다.

- 변수는 메소드 블록 내 어디서든 선언할 수 있지만, 변수 사용은 제한이 따른다.

  ```
  변수는 선언된 블록(Scope) 내에서만 사용이 가능하다.
  ```


### 2.2 데이터 타입

- 모든 변수에는 타입이 있으며 타입에 따라 저장할 수 있는 값의 종류와 범위가 달라진다.
- 변수의 타입은 선언 후 변경 불가능.
  - 실행 중에 저장할 수 있는 값의 범위를 초과하면 최소값부터 다시 반복 저장된다.
    - byte : -128~127이고, 이후 128이 아닌 다시 -128부터 시작한다. 즉, 범위를 초과하지 않게 사용해야 한다. (아니면 쓰레기값  들어감)
- 자바는 유니코드 사용.
  - 유니코드 : 0 ~ 65535, char : 0 ~ 65535 (2^16)
- 기본(primitive) 타입
  - 정수
    - byte(1byte) : $-2^{7} \text{~} 2^{7}-1$ 총 $2^8$(1byte)
    - char(2byte) : $0 \text{~} 2^{16}-1$
    - chort(2byte)
    - int(4byte) : 정수. **byte나 short를 사용하여 연산을 하더라도 어차피 int로 변환하여 수행함.**
    - long(8byte)
  - 실수
    - float(4byte)
    - double(8byte)
  - 논리
    - boolean(1byte)

### 2.3 타입 변환

데이터 타입을 다른 타입으로 변환하는 것

- 자동(묵시적) 변환 : Promotion
  - **작은 타입 $\rightarrow$ 큰 타입**으로 자동 타입 변환 가능
  - `byte(1) < short(2) < int(4) < long(8) < float(4) < double(8)`
```java
int intValue = 10;
double doubleValue = 5.5;
double result = intValue + doubleValue; // result = 15.5, intValue가 double 타입으로 자동 변환됨.
```
- 강제(명시적) 변환 : Casting
  - **큰 타입 $\rightarrow$ 작은 타입**으로 강제 타입 변환 가능

```java
int intValue = 201502119;
byte byteValue = (byte) intValue; // int형을 byte형으로 명시적 변환
```
