## 용어

- **릴레이션 (relation)** 

같은 성격의 데이터들의 집합을 의미. 흔히 테이블이라고 말하는 용어와 같은 의미로 이론적인 용어. 

릴레이션은 튜플과 에트리뷰트로 데이터를 정렬하여 관리한다. 



- **튜플 (tuple)**

릴레이션의 각 `행`을 의미. 

흔히 일반적인 용어로 `레코드`(record)와 `로우`(row)와 같은 의미로 사용된다. 



- **에트리뷰트(attribute)**

릴레이션에서 이름을 가진 하나의 `열`을 말한다. 흔히 일반적인 용어로 `칼럼`(column)과 같은 의미로 사용된다. 



- **디그리(degree)**

에트리뷰트의 수를 말한다. 즉, `칼럼` 의 수



- **카디널러티(cardinality)**

튜플의 수를 말한다. 즉, `로우` 의 수



## HINT

#### char vs varchar

| char(20)                                              | varchar(20)                     |
| ----------------------------------------------------- | ------------------------------- |
| **시간** good                                         | **공간** good                   |
| 고정형 : 'Hi' 넣어도 20byte 차지                      | 가변형 : 'Hi' 넣으면 2byte 차지 |
| 개빠름. 사이즈 고정일 때 효율적. (주민번호, 아이디..) | 상대적 느림                     |

[참고 자료](https://121202.tistory.com/26?category=541709)

```mysql
SELECT * INTO [새테이블] FROM [복사할테이블] WHERE 1=2 # 테이블 구조 복사
INSERT INTO [테이블] VALUES (val1, val2, val3)
UPDATE [테이블] SET [열] = '변경할값' WHERE [조건] is not null
DELETE FROM [테이블] WHERE [조건]
```

> SELECT 문에서 INTO가 들어갈 경우 조건에 맞는 기존 테이블의 내용을 가져와 새 테이블로 만든다.
