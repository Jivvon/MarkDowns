####정보표현의 단위

bit > byte > word > field > record > file > database

#### Database System

- 사용자
  - DBA (Database Administrator)
  - Application Programmer
  - End User
- 데이터베이스의 스키마(intension)와 상태(extension)는 분리되어 저장됨

#### Database Design

개념적 모델링 -> 논리적 모델링

#### Database 사용 분야

- Data mining : 지식창고
- Data warehousing : 통합적 환경
- Data marts : 부서중심의 데이터 웨어하우스 (더 작은 개념)

#### DBMS 발전 과정

![스크린샷 2020-03-24 오후 8.33.30](../images/스크린샷 2020-03-24 오후 8.33.30.png)

- 계층 DBMS
  - IBM사의 **IMS** : 트리 구조
- 네트워크 DBMS
  - 하니웰의 **IDS** : 노드(레코드) + 간선
- 관계 DBMS
  - IBM 연구소의 **System R**과 버클리대의 **Ingres** 프로젝트 : 오라클, MS SQL Server 등
- 객체 지향 DBMS
  - 장점 : 데이터와 프로그램을 그룹화하고, 복잡한 객체들을 이해하기 쉬우며 유지와 변경이 용이함
