## Software Design Pattern

### Sortware Design pattern

- **Creational patterns**
  : `Object Pool` : 미리 생성해놓고 쓰고 반납하고, **`Singleton`**, Abstract Factory, Prototype

  ```
  Singleton

  목적 : 공통 엔진 하나. 공통 state 하나. 즉, 별칭은 같을 수 있어도 적용되는 인스턴스는 하나이다.

  목적과 의도상에 있어서 글로벌 변수이다. but, 글로벌 변수처럼 쓰면 안 됨.
  ```

- **Structural patterns**
  : `Adapter`, **`Decorator`** : 앞 뒤로 붙여서 또 다른 기능을 할 수 있도록, **`Facade (파사드)`** : 하나의 개념적인 창 ? private class Data, proxy

  ```
  Decorator

  원본은 그대로 있고 거기에 붙여나가는 개념으로 decoration을 추가한다.
  ```

- **Behavioral patterns**
  : `Antipattern`, `Observer`, Iterator, Interpreter, Command

### 정적메소드 ( @staticmethod vs @classmethod )

```
결정적 차이는 상속에서 나타난다.

static method는 가장 위로 올라감 (최대한 부모)

class method는 일반적인 상속
```

**static method**

```python
class A:
    def __init__(self):
        pass

    # def sum(self, a, b):
    #     print(a+b)

    @staticmethod
    def sum(a, b):
        print(a+b)
```
