## OOP의 장점 / 단점

class : 추상화 시켜놓은 개념
object : class를 구체화 시켜나간다.

## **`OOP 개중요`**

    - Classes
        속성과 행동을 하나로 들고 다니자.
        blueprint or pattern 과 같이 생각할 수 있다 - 마치 프로그래밍의 설계도처럼
    - Inheritance 계층
        - Single            A - B
        - Multiple          [A / B] - C
        - Multilevel        A - B - C
        - Hierarchical      A - [B / C / D]
        - Hybrid            A - [B / C] - D
    - Polymorphism : 같은 이름을 가진 서로 다른 구현체가 있을 수 있다. (추상화 한 것을 객체마다 다르게 구체화한다)
    - Data Abstraction
    - Encapsulation : 관계 있는 것들은 묶자(Modulization) + 노출할 필요가 없는 것들은 숨긴다(Information hiding)
    - Information hiding : 노출할 필요가 없는 것들은 숨긴다

**`Inheritance`**
UML에서는 방향이 반대다
B depends on A : from B to A

## **`Encapsulation과 Information hiding 의 차이`**

    Encapsulation = Modulization + Information hiding

class라는 개념을 인스턴스화 - **`메모리에 있느냐 없느냐`**
